## What was NOT shipped from Stitch's output

- **`CARD SHED` brand caption in the header.** Stitch added a small Playfair Display caption between the back link and a right-side spacer. Removed ‚Äî the h1 "WHO'S PLAYING?" already anchors the page, and adding a second occurrence of the brand violates "single h1 per route" + creates redundant chrome the user doesn't need on a pre-match screen.
- **Material Symbols Outlined font load.** Stitch wired `<link>` tags for the icon font twice and used `arrow_back_ios` + `arrow_drop_down`. Replaced with Unicode glyphs (`‚óÄ`, `‚ñº`, `‚ñ≤`) to match the S01 "no Material Symbols until S09/S10" decision ‚Äî keeps page weight down and removes one CDN dependency.
- **`¬© 2024 CARD SHED v1.0.4-beta. Tactical Precision Engine.` footer line.** Stitch added an ornamental right-aligned footer caption. Removed ‚Äî same reason as S01's `v1.0.4 Tournament Mode Active` line: it's ornament implying state we don't actually have.
- **`overflow: hidden` on `<body>`.** Stitch's inline `<style>` clamps body overflow. Skipped ‚Äî at 1440 √ó 900 the page already fits; at narrower viewports we want natural scroll, and global `overflow:hidden` would break that.
- **Inline `tailwind.config`.** Same as Run #2 ‚Äî DS-1's `@theme` block in `apps/ui/src/index.css` already provides every token the screen needs.

## Engine coupling decisions for M4

- **Seed source:** `crypto.getRandomValues(new Uint32Array(1))[0]`, called exactly once per Randomize click. Located in `apps/ui/src/state/matchStore.ts` as `generateMatchSeed()` and re-used as the `PlayerSetup` default. The core itself never references `crypto` ‚Äî the rule from `.claude/rules/core-determinism.md` is preserved by construction.
- **`matchStore`:** zustand store with shape `{ match: MatchState | null, startMatch, reset }`. `startMatch({ players, seed })` synchronously calls `@cardshed/core.startNewRound(null, seed, { matchId, players })`. `matchId` is `m-<seed-hex>`.
- **Persistence:** intentionally NOT wired. PRP 3 M4 warned about quota risk if MatchState lands in localStorage on every action. The agreed cadence is "throttle to round boundaries", which becomes observable at M7+. Holding the feature back until then.
